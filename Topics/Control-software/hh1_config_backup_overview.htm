About Backing Up and Restoring System Configurations

See Also for links to information about backing up and restoring system configurations.

Purpose: The backup function creates and manipulates backups of system configuration information. You can:

● Restore previously created configuration backups
● View and edit information about each configuration backup
● Add or remove configuration backups
● Save configuration backups to external devices
Access: On any CONFIGURATION screen, tap the BACKUP AND RESTORE tab.

Note:  You can also restore a backup that has been saved onto an external memory device (DVD, CD, or flash drive).

Caution!  Restoring a backup will overwrite all current control system configuration information.

Note:  You must have supervisor permissions to restore a configuration backup.

A configuration backup contains all system settings, user profiles, alarm levels, and device layouts. Historic data like temperature or compensator history is not saved.

Backups types are:

● Manual. You can make a manual backup of the system configuration at any time.
● Automatic. The control system makes an automatic backup just before you restore a previously saved backup and just before you make changes to a node (for example, adding or removing a node).
