Device Interface Categories

See Also for links to information about signal processing and device interfaces.

Devices on the ROV and TMS are connected to the DTS control system through cables attached to subsea node ports. Each node port contains an interface module that routes signals between the device and the surface monitoring equipment and controls.

Devices can generally be categorized by:

● Signal type—whether format for the device signal is:
— Serial
— Video
— Ethernet
● Signal processing location—whether the device’s signals are sent to and processed by:
— The real-time computer (RTC). These devices have a “system interface" because their signals are processed by the ROV system.
— The customer’s computer or other customer-provided equipment. These devices have a “user interface."
● Preconfiguration status—whether configuration information for that devices is preloaded into the control system. This determines whether a device is preconfigured or custom.
— Common models of common ROV devices (like navigation equipment, cameras, and lights) are available for you to pick from a list. Any item on that list is a preconfigured device, and is preconfigured for use by the control system. A preconfigured device can have either a system interface or a user interface.
— Equipment that is not on the preconfigured list must be installed as a custom device. You must configure custom devices with information supplied by the equipment manufacturer. All customer devices have a user interface.
● Signal duplication (called mirroring)—when a device's signals are duplicated and sent to two locations. The device can receive control signals from only one of the two locations. The device must also connect to node ports that contain a bicast SIM (serial interface module).
