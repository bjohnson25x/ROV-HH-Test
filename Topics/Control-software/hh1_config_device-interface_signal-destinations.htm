Signal Processing Locations

Signal Processing Locations
Device Signal Destination Description Example
System interface The device sends communications through the DTS network to the RTC for processing by the ROV control system. Device data is available on control system screens and the device is controlled by ROV system software. Most actuators, tools, navigation equipment, lights, and sensors have a system interface. For example, a light that is controlled through the LIGHTS panel on the FLIGHT screen has a system interface.
User interface

In a process called “user tunneling,” the device sends serial communications through the DTS network but does not communicate with the ROV control system. Data is transmitted from the device to a rack node port, and from there to a customer’s external computer or other processing equipment for customer use. The device is then operated from the external computer rather than the ROV control system.

For user tunneling to work, the customer’s computer or processing equipment must be attached to the rack node that is communicating with the device.

Includes preconfigured devices that will be operated by an external computer like a DVL used by a survey system.

Includes all custom devices (like a navigation instrument that is not on the list of preconfigured devices).

Signal Mirroring (Duplication)

In mirroring, a device's signals are duplicated (that is, device signals are split into two separate, identical signal sets).

Mirroring Used with a System Interface

Mirroring is generally used with devices that have a system interface (that is, they communicate with the RTC). In this arrangement, one signal set is sent to the RTC for processing by the ROV control system, and the other is sent ("mirrored") to a rack node port where it can be accessed by the a customer computer or other customer processing device.

In an example of mirroring for a device with a system interface, DVL signals are sent to the RTC for use in StationKeep and other automatic control modes, while a duplicate set of DVL signals is simultaneously sent to a customer’s survey computer.

Mirroring Used with a User Interface

Devices that use a serial user tunnel (where processing occurs in customer equipment instead of the RTC) can also be mirrored. (However, mirroring is much less common for devices with a user interface than for those with a system interface). For mirroring of a device with a user interface, each of the two identical signal sets is communicated through a rack node port. However, only one of the signal sets can control the device.
