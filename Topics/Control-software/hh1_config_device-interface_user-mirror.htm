Device User Interfaces and Mirroring Compared

See Also for links to information about signal processing and device interfaces.

In both user interfaces and mirroring, device signals are sent to a rack node for access by an external computer or other processing device. However, user interfaces and mirroring are typically used by different device types and for different purposes:

● In user interfaces, the device is controlled not by the ROV control system but by customer-supplied equipment that communicates with the device through the user tunnel.
● In mirrored interfaces, the device is typically controlled by the ROV control system, but a duplicated data stream from the device is sent through a rack node port to customer-supplied equipment for data analysis or logging.

Note:  When you create a user interface (with its user tunnel) or a mirror to a rack node port, the ROV control system reserves that port so that it can’t be used for anything else as long as the user tunnel or mirror exists. If you remove the device from ROV system, the rack node port for the tunnel or mirror is released for other uses.

Configuring a User Tunnel

User tunnels are established when you add a device to the ROV system (see Installing a Preconfigured Device and Installing a Custom Device). At that time, you can pick a rack node port for the user tunnel.

Configuring a Mirror Port

Mirror ports are established after you have already added the device.

1. On any CONFIGURATION screen, tap the DEVICE tab.
2. Tap EDIT CONFIGURATION.
3. In the EDIT CONFIGURATION panel, tap the MIRROR PORT tab.

See MIRROR PORT Configuration Tab for additional steps.

Creating a Temporary Test Tunnel

If you need to run test software for a device with a system interface, you can temporarily reroute device signals from the ROV control system and send the signals through a test tunnel to a rack node port. For more information, see Port POWER AND COMMUNICATION Diagnostics Panel.
