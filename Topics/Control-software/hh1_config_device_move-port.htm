Moving a Device to a Different Port

See Also for links to general information about configuring devices.

It is usually best to change a deviceâ€™s physical node port connection before you change the port in the control system software.

Caution!  The control system cannot check to ensure that you have connected a device to a compatible interface module on a node port. (For example, the control system does not detect the connection of a device that sends serial signals to a port with a video interface module.) If you do not accurately match port hardware with the configuration information about that hardware, equipment may malfunction.

Note:  The procedure in this section applies to preconfigured devices with a system interface (that is, devices that do not have a user tunnel or a mirrored interface). To change the location of a rack node port for a device with a user tunnel or a mirrored interface, you must remove the device and then reinstall it with the desired rack node port.

1. Ensure that the user profile for your control station includes supervisor permissions.
2. Display any CONFIGURATION screen.
3. Tap the DEVICE tab at the top of the screen.
4. Tap the tab for the correct device category
5. From the drop-down list, tap the button for the device.
6. Tap CHANGE PORT at the bottom of the screen.

7. When the MOVE DEVICE TO NEW NODE PORT panel displays, display the drop-down list of available (open) ports that have appropriate interface modules.

Note:   You cannot move a device to a different type of interface module; for example, you cannot move a device from a SIM port to a VIM. Also, you cannot move a device to a node on a different platform; for example, you cannot move a device from a port on an ROV node to a port on a TMS node.

8. Tap the button for the desired port.
9. Tap MOVE DEVICE.
