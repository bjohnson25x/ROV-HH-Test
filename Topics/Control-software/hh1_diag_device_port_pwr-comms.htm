The POWER AND COMMUNICATION panel displays on the diagnostics screen for any device that attaches to a node port. This panel lets you:

● Turn port power on and off.
● View status of port current and software fuses.
● View status of port Ethernet traffic.
● View status of the port's communication with the RTC.
● Establish a temporary communication tunnel to test the device.

Port Power

In the POWER AND COMMUNICATION panel for an individual device, you can turn port power on and off for that device.

Note:  You can also turn port power on and off from the DEVICE POWER panel on the FLIGHT screen.

Port Current and Software Fuses

In the POWER AND COMMUNICATION panel for an individual device, the current status display shows current in 0.1 A.

See details

The color of the software fuse indicator shows fuse status:

● Green = enabled fuse
● Gray = no power
● Red = fuse fault
● Orange stop sign = fuse has been disabled

When a software fuse trips, port power automatically turns off. Turning power back on resets the software fuse.

You can enable or disable software fuses in the POWER tab for device configuration (see Enabling and Disabling Device Software Fuses).

Port Ethernet Traffic

In the POWER AND COMMUNICATION panel for an individual device, the IN linear gauge indicates Ethernet traffic flowing out of the port’s interface module and into the Ethernet switch. The OUT gauge indicates traffic flowing out of the Ethernet switch and into the port. This means that IN and OUT are relative to the node switch rather than to the port’s interface module. The gauge indicators and text numbers represent a percentage of full scale rather than megabits per second.

See details

Normal expected values depend on the type of module installed:

● When a client/server VIM pair is functioning correctly, the IN bar for the subsea node (server) VIM or the OUT bar for the rack node (client) VIM will generally be at full scale.
● When device connections to SIMs and NIMs are functioning correctly, both bars will generally be around one-quarter to one-third of full scale, depending on the connected device.

Port RTC Communication Status

In the POWER AND COMMUNICATION panel, port RTC communication status information displays for devices with a system interface (that is, for ports that communicate with the RTC). For devices with a user interface (that is, those with a user tunnel), RTC information does not appear.

See details

Status of RTC communications is shown by color of the indicator:

● Green = OK
● Orange = failure rate is at warning level
● Red = failure rate is above alarm threshold
● Gray = no port power

As the RTC sends commands to the device at the selected port, the ATTEMPTED field shows a series of white dots. The SUCCEEDED field shows white dots as the device that is connected to the port responds to the RTC. These fields are not in units of any type; they indicate only whether communication is occurring.

The white dots in the ATTEMPTED and SUCCEEDED fields may become misaligned as they move; this does not indicate any problem.

The FAILED field shows the percentage of RTC communication attempts to which the device did not respond. During normal communications, the FAILED value is 0.0%.

Occasionally a device produces a few errors at startup, resulting in a small percentage in the FAILED box. If no further errors are produced, the value should drop to 0.0% as the number of ATTEMPTED and SUCCEEDED communications increases over time. A persistent or rising failure rate indicates a communication problem.

To reset the FAILED value, long press the screen area containing communications information and tap CLEAR COMMUNICATIONS STATISTICS.

Port Test Mode

Use the PORT TEST MODE button in the POWER AND COMMUNICATION panel to create a temporary tunnel for testing a device attached to the ROV system. For more information, see Port Test Mode.
