Port Test Mode

See Also for links to more diagnostics information about node ports.

Use the PORT TEST MODE button in the POWER AND COMMUNICATION panel to create a temporary tunnel for testing a device attached to the ROV system.

Typically, devices with a system interface (including most actuators, tools, navigation equipment, lights, and sensors) send communications through the DTS network to the ROV control system for processing through the real-time computer (RTC). If you need to diagnose a device problem by running test software, you may want to disengage that device from communicating with the RTC and direct device signals to a computer running test software that can diagnose the device's problems.

You can either chose an available rack node SIM port for testing/diagnostics purposes or let the control system select an available port for you.

To let the control system select the port for you:

1. Tap PORT TEST MODE. The button becomes active and shows the node and port number for the temporary test tunnel.
2. Attach a computer to the rack node port and run the device test software.
3. When you finish testing the device, tap PORT TEST MODE to make the button inactive. This returns the device to normal communications with the RTC.

To select the port yourself:

1. Tap the down arrow on PORT TEST MODE.
2. Select a port from the drop-down list of available rack node SIM ports.

3. Tap PORT TEST MODE to make it active.
4. Attach a computer to the rack node port and run the device test software.
5. When you finish testing the device, tap PORT TEST MODE to make the button inactive. This returns the device to normal communications with the RTC.
