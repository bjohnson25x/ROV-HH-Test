Issue Types

See Also for links to information about issues and events.

Issues are categorized (and color coded) according to their level of severity (that is, their potential to harm ROV system equipment or operations).

Issue Severity
Issue Category Meaning Color Code Example
Information Something significant has happened Yellow

A tactile control has been assigned to console station
or
The ROV has docked to the TMS
Warning A situation likely needs attention Orange A compensator’s fill level is below the its configured low-fill threshold
Alarm A situation needs urgent attention Red Water is detected in the ROV IPDS or IPA

Issues are also defined by other characteristics, as described in the table below.

Issue Characteristics
Description Definition Other Information
New issue The operator has not yet acknowledged that he or she has seen the issue notification Issue can be either active or cleared
Acknowledged issue The operator has acknowledged that he/she has seen the issue notification Appears on the issues list as long as the issue is active; disappears from the issues list as soon as the issue clears
Active issue The issue continues to be outside of normal conditions or operational ranges Can be either new or acknowledged
Cleared issue The issue has returned to normal conditions or operational ranges Appears on the issues list as long has the issue is unacknowledged; disappears from the list as soon as the issue is acknowledged

Note:  You can acknowledge an issue only from the Status Bar issues list. You cannot acknowledge an issue from the issues log or from the list of issues for each platform on the DIAGNOSTICS dashboard.
