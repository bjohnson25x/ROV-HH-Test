Communication Path Status

See Also for links to information about telemetry diagnostics.

From the DIAGNOSTICS dashboard, select a telemetry subsystem (like the ROV or TMS) to display information about network paths.

Possible Communication States for Paths Between Nodes
State Description
Normal The path is able to send and receive signals.
Lost The path is unable to send and receive signals.
Recovering Communications were previously lost but have been recently restored. A recovering path is temporarily given lower priority by the RSTP system. After a set time has passed without lost communications (see the section on RSTP in Power, Telemetry, & Compensation Systems), a recovering path is given “normal” status.
Locked out The operator has specified that the path should be given lower priority for selection by the RSTP system. The path can still be used if no better path is available.
Active Currently selected for use by the RSTP system.
Inactive Not currently selected for use. Even if a path is fit for use (has a “normal” status), it can be inactive if the RSTP system currently prefers an alternate path.

A recovering path and a locked-out path have the same low priority for being chosen. They differ only in that the control system designates a path as recovering while the operator designates a path as locked out.
