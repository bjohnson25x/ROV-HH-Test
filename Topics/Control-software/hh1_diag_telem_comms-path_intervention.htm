Operator Intervention in Communication Paths

See Also for links to information about telemetry diagnostics.

The diagnostics screen for telemetry information provides tools for managing communication paths.

Assigning a Path Low Priority

You can tell the RSTP system to assign a path the lowest priority for selection. If a path’s communications have been intermittent and you believe the path to be undependable, locking it out will generally increase communication efficiency. However, locking out a path is simply your recommendation to RSTP and is ignored if the control system must use the locked-out link because no better path is available.

To lock out a path:

1. Select the desired path.
2. Tap the drop-down box for path tools.
3. Tap LOCK OUT.

Note:  If you frequently power the TMS on and off during ROV operations, you can lock out the connection between the TMS and ROV to prevent disruption to ROV communications.

Forcing Path Recovery

You can also force a path from the “recovering” state to the “normal” state. If, for example, you turn off a node and then turn it on again, the communication system initially designates paths to that node as “recovering” just because the paths were unavailable while the node was off. If you are confident that a path is sound, you can force the system to remove the “recovering” status so that the path no longer has a low priority. However, if the control system detects that the path is not reliable, it will return the path to the “recovering” state.

To force path recovery:

1. Select the desired path.
2. Tap the drop-down box for path tools.
3. Tap FORCE RECOVERY.

Note:  When you first power on the ROV or TMS, you should force the recovery of any failed paths.
