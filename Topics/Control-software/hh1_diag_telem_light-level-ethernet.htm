Light Levels and Ethernet Traffic

See Also for links to information about telemetry diagnostics.

Select a telemetry subsystem (like the ROV or TMS) to display the screen showing light levels and Ethernet traffic.

Note:  For Ethernet traffic levels, values shown on the gauges and text fields are relative and not in absolute units (like mBps).

Each node has two SFP modules. Each module transmits data on one of two wavelengths and receives data on the other wavelength. SFP modules with a blue label transmit on 1310 nm and receive on 1550 nm, while SFP modules with a green label transmit on 1550 nm and receive on 1310 nm. Each node has one of each type of module.

When you make fiber connections between nodes, the blue module (designated module A) of one node must be connected to the green module (designated module B) of another node, and vice versa. If two nodes are connected using modules of the same color, they will not communicate.

In the Digital Telemetry System, a node’s blue SFP module connects to the next node upstream in network (toward the surface). A node’s green SFP module connects to the next node downstream in the network (toward the seafloor).

The default alarm threshold for low light level is -28 dBM. Setting the pointer to less than -30 disables the alarm.

In the CONNECTION DETAILS screen, “time until recovery” is the time remaining before a path that experienced a failure is considered to be sound again.
