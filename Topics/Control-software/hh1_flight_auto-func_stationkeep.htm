STATIONKEEP Features
Purpose

Keeps the ROV at designated starboard/port and fore/aft coordinates. (The ROV can still move up and down, pitch and roll, and change heading.)

Mid-Water StationKeep (optional feature) performs the StationKeep function when the ROV is not near the seafloor.
Access On the FLIGHT screen, tap STATIONKEEP.
Potential issues

The DVL must be within range of the seafloor for StationKeep to work.

If you select LOCK CONTROLS, flight joystick input for fore, aft, starboard, and port movement are locked out. Vertical movement, pitch/roll, and yaw input are still allowed, but pitch and roll must be commanded from the TRIMS function or AUTO PITCH/ROLL.

Caution!  STATIONKEEP holds the ROV stationary only for fore/aft and starboard/port movements. To keep the ROV from moving at all, STATIONKEEP, AUTO DEPTH or AUTO ALTITUDE, and AUTO HEADING must all be on.
Data input
● Compass plus motion reference unit (MRU) or MRU with heading capability.
● DVL.
Turning on/off
● Screen STATIONKEEP button.
● Flight control pod button.
Initial set point The ROV’s fore/aft and starboard/port position at the time when you turn on StationKeep.
Changing the set point
● Tap PORT, STBD, FORE, or AFT.
● If the LOCK CONTROLS button is not active, you can use the joystick to move the ROV to a new location, and then release the flight joystick to its neutral position. The ROV’s location at the time when you release the joystick becomes the new set point.
Function interactions
● Not available in BOTTOM MODE.
● Not available in PARK mode.
● You can change the STATIONKEEP position using the DISPLACE function.

Mid-Water StationKeep

Note:  Mid-Water StationKeep is available only if a license for this feature has been installed on the ROV system (see Installing and Viewing Software Feature Licenses).

Mid-Water StationKeep performs the StationKeep function when the ROV is not near the seafloor. To use Mid-Water StationKeep:

● The ROV must be equipped with an inertial navigation instrument along with a DVL that provides input to that instrument. (The inertial navigation instrument and the DVL may be packaged in a single assembly.)
● The ship must be equipped with an ultra-short baseline (USBL) transceiver acoustic positioning system that provides data for Mid-Water StationKeep.

The quality of STATIONKEEP data determines how accurately STATIONKEEP performs. Mid-Water STATIONKEEP is most accurate if the inertial navigation instrument/DVL and USBL are providing input to (aiding) STATIONKEEP, but can also function if only the inertial navigation instrument/DVL or only USBL is aiding the process. Items in the STATIONKEEP panel indicate whether Mid-Water STATIONKEEP is being aided by the inertial navigation instrument/DVL, by USBL, or both.

The control system generates a warning if Mid-Water StationKeep becomes unaided by either the inertial navigation instrument/DVL or USBL.

Mid-Water StationKeep Indicators

The quality of the data provided to Mid-Water StationKeep is indicated by a screen gauge. When data quality falls below a defined threshold (which you can configure), StationKeep automatically disengages. You cannot engage StationKeep again until the data quality rises above the threshold.

You can view Mid-Water StationKeep data indicators in the STATIONKEEP panel.
