Survey Software and Hardware for AutoTrack

See Also for links to AutoTrack (survey operations) information.

The AutoTrack feature requires that customer-supplied hardware and survey software be connected to the ROV control system.

The survey software, NaviPac version 3.6 P6 or later, receives data from the survey system, examines it for validity, and passes it to the ROV control system. To interface the NaviPac software with AutoTrack, you must add a new data output (set to Schilling Autofollow) in the NaviPac setup application.

Note:  This NaviPac "Schilling Autofollow" data output is used for AutoTrack. It should not be confused with the ROV control system's FLIGHT screen AutoFollow function, which lets a pan and tilt follow manipulator arm movement.

For additional information on NaviPac software, see the OEM manual located in Support Files\3 - OEM Docs\Survey package\.

Navipac Autofollow Output String
String Element Description
Output string is: $NAVIPAC,<XTE>,<L/R>,M,<Dir>,<Position Source>*<checksum><cr><lf>
$NAVIPAC

Indicates the beginning of a new string.
XTE The cross-track error. This variable uses only metric units with a significant number of digits (for example, 12.345).
L/R

Directs the vehicle to move right (R) or left (L) to hit the line.
Dir If a straight line, the current line direction; if a curved line, the current tangent to the line. Measured in degrees, and the ROV steers to this heading.
Position Source

NaviPac's position data source. If the value is -1, the output string is treated as invalid.
Checksum Calculated using NMEA rules, starting at N and ending at <Position Source>.
cr Carriage return.
Lf Line feed.

 

Navipac Autofollow Output String
Position Value Description
POS_UNDEF -1 Invalid data
POS_SURFACE 0 HIPAP (High-Precision Acoustic Positioning)
POS_HPR 1 Raw USBL (Ultrashort Baseline)
POS_LBL 2 LBL (Long Baseline)
POS_DVL 3 USBL combined with Doppler log
POS_HAIN 4 Hain-based USBL
