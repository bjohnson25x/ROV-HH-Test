Schilling Survey Input String for HDEVS Video

See Also for links to information about configuring data inputs and outputs.

Note:  This information applies only to ROV systems that have HDEVS rather than TAVS.

The Schilling survey data input string is used to transmit information from survey equipment to the control system so that the data can be used in video overlays. The data (including items like kilometer post [KP]) is input from the survey equipment through the serial expander to the control system.

To use the Schilling survey input string, you must first add the string to the control system. You must also configure the input by specifying an input port and data format.

The Schilling Survey input is in the form of this comma-delimited string:

$SURVEY,<Date>,<Time>,<Northing>,<Easting>,<KP>,<DOL>,<Heading>,<ROV Depth>,<ROV Altitude>, <Custom1>,<Custom2>*<Checksum(opt)>#CR#LF

The Schilling survey string must begin with a dollar sign character ($) and end with carriage return and linefeed.

The Custom 1 and Custom 2 items can be any string of characters that you wish to use in video overlays. The character string cannot include a comma because a comma indicates the end of a data item.

The Checksum item in the string is optional. However, if you use the Checksum, you must precede it with an asterisk (*) character.

Example string: $SURVEY,08182014,1245,4807.038N,2231.000E,123,100.3,3001,10.2,3462.7,CUST1,CUST2
