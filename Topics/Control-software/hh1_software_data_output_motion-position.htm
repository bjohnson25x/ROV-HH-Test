Exporting the Data

The control system collects information about ROV movement and position, and that output is available for external use (for example, with survey software) through a port on the serial expander unit. See the ROV system interconnect drawing for information about equipment connected to the serial expander.

Use a cable/connector combination (like RJ-11 to DB-9), to input the data into a patch panel. Pins for survey string output on a serial expander RJ-11 port are:

● Pin 1, Receive
● Pin 2, Signal Ground
● Pin 4, Transmit

Data String Format

Data is provided at 2 Hz using a comma-delimited string as shown in the example below:

HDRov, 180.2, 02555.4, 060.6, 045.6, 1, NOCP<CR><LF>

This data string is sometimes called a survey string because it can be used by survey software.

Each string starts with letters and/or numbers identifying the ROV system type and ends with <CR><LF>. The items in between are data provided by navigation instruments like the MRU, compass, and depth sensor.

The string reports the following information in the following order:

1. ROV type (for example, “HDRov” indicates an HD model ROV)
2. Heading in degrees
3. Depth in meters
4. Altitude in meters
5. Pitch in degrees
6. Roll in degrees
7. Telemetry on/off (1 for on, 0 for off)
8. Comment (typically says “NOCP,” indicating no comment)

Port Configuration for Receiving the Data

In the device that receives the data (typically a survey computer), the receiving port must be configured with the following serial communication parameters:

● Baud rate: 9600
● Data bits: 8
● Parity: none
● Stop bits: 1
● Handshake: None
