Creating a Video Diagnostic Snapshot

See Also for links to information about the Video Computer App. The Video Computer App is used only with HDEVS.

This information applies only to ROV systems that have HDEVS rather than TAVS.

The diagnostic snapshot feature creates a file that contains information for use by factory technical support personnel in diagnosing problems. A support technician will tell you when this file is needed.

Note:  You can also take a diagnostic snapshot from one of the ROV control system computers (see Console Subsystem Diagnostics Display). However, if you are experiencing problems with a video function (for example, a video computer is not recording video), you should take the diagnostic snapshot from the Video Computer App.

You can save either a basic (partial information only) or complete snapshot. The complete snapshot contains more information, but it may be too large to e-mail.

To use the video diagnostic snapshot function:

1. Determine which video computer is not functioning correctly.
2. Insert an external memory device into a USB port in that video computer.
3. Go to the VIDEO CONFIGURATION screen (see Configuring Recording Details and Viewing Overlays).
4. Click the video diagnostic snapshot icon.
— To create a basic snapshot, left-click the icon.
— To create a complete snapshot, right-click the icon and then click Complete System Snapshot.

5. When the SELECT FILE SAVE LOCATION box displays, navigate to the desired external memory device.

6. If you wish to change the default file name for the snapshot file, click the name and type a new one. The file name must have the extension .srs.
7. Click SAVE.
8. Safely eject the external memory device and send the .srs file to a technical support representative.
