Video Computer App Offline Mode

See Also for links to information about the Video Computer App. The Video Computer App is used only with HDEVS.

This information applies only to ROV systems that have HDEVS rather than TAVS.

To record video streams from ROV system cameras, the video computer must communicate with computers for the ROV control system (the client computers and the RTC). When the video computer cannot communicate with the control system (for example, because the control system has been restarted), the Video Computer App goes into an offline mode and can no longer preview or record video streams.

When the Video Computer App is offline, you can still:

● Edit overlay layouts and save layouts on an external memory device.
● Play back video segments that have already been recorded and export those segments to an external memory device.
● Set options on the VIDEO CONFIGURATION screen.

Offline Mode Conditions and Fixes

The Video Computer App can be in offline mode for any of the conditions described below. When these conditions occur, an error message displays in the video computer information section of the Video Computer App DASHBOARD.

Problem Condition Possible Cause Steps to Fix
No communication at startup When the video computer started up, the ROV control system had not started up (or startup was not complete). When the ROV control system fully starts up, the Video Computer App automatically exits from offline mode and becomes fully operational.
Lost communication The video computer established and then lost communication with the ROV control system (for example, because the ROV control system was restarted). Shut down and then restart the video computer.
Mismatched software versions The software version for the Video Computer App does not match the software version for the ROV control system computers. Load the appropriate software version for the Video Computer App.
